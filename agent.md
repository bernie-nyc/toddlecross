# Mnemosyne — Operational Policy

This file is the authoritative source for project conventions. AI agents working on this repository **must read it before making changes and must follow the rules below**. Humans should treat it as the binding equivalent of `CONTRIBUTING.md`.

When project conventions change, update this file first and reference it elsewhere — don't duplicate the rules in scattered places.

---

## Branching

This repo uses **two long-lived branches**:

| Branch | Purpose | Who writes to it |
| --- | --- | --- |
| `main` | The good code. Deployable at all times. | **Only** via PR from `dev`. Never direct. |
| `dev` | Integration branch. All work happens here. | Feature branches PR back into it. |

### Rules

1. **Never commit directly to `main`.** Every change to `main` arrives as a PR from `dev`. Branch protection on `main` enforces this.
2. **Feature branches branch off `dev` and PR back to `dev`.** Naming: `feat/...`, `fix/...`, `chore/...`, `docs/...`, `style/...`, `migration/...`. **Branch names must never contain "claude", a model name, or any Anthropic/AI-service identity.** If a session starts on an auto-generated branch (e.g. `claude/<random-name>`, which Claude Code on the web creates by default), rename it to a convention-compliant name before opening a PR, or branch off it into a compliant name and PR from that.
3. **After every merge of `dev` into `main`, `dev` is fast-forwarded back to `main`.** The GitHub Action at `.github/workflows/sync-dev-to-main.yml` does this automatically on every push to `main`. If it ever fails — meaning `dev` has diverged from `main` — that means someone committed directly to `main` (emergency hotfix, GitHub UI edit, etc.) and `dev` needs a manual reconciliation:
  
  ```bash
  git checkout dev
  git fetch origin
  git merge --ff-only origin/main   # if this fails, see below
  git push origin dev
  ```
  
  If `--ff-only` fails because `dev` has commits not in `main`, resolve by merging `main` into `dev` (regular merge, not FF) and pushing the merge commit. **`dev` is never allowed to be behind `main`.**
4. **Hotfix-direct-to-main is reserved for emergencies.** Acceptable cases: a production-down incident where waiting for a PR cycle is unsafe, or a one-line edit via the GitHub UI (typo in a doc, etc.). Every such case immediately triggers the resync Action; if the Action fails, the human who made the hotfix is responsible for fixing `dev` before any other work proceeds.
5. **Delete feature branches immediately after their PR is merged.** No branch should outlive its merge. Enable "Automatically delete head branches" in repo Settings → General so GitHub handles this on every PR merge. AI agents must not leave session branches, renamed session branches, or any `feat/`, `fix/`, `chore/`, `docs/`, `style/`, or `migration/` branch on the remote after the PR is closed.

### Default behavior for AI agents

- **Default working branch is `dev`, not `main`.** When a session starts on `main`, switch to `dev` before making changes unless the user explicitly says otherwise.
- **When the user says "commit and push" without specifying a branch:** push to the current feature branch, or to `dev` if working there. Never push to `main`.
- **Open PRs against `dev` by default.** The only PR that should target `main` is a periodic `dev` → `main` promotion, or an emergency hotfix.
- **Never use `git push --force` on `main` or `dev`.** Force-pushing to a feature branch you own is fine.
- **Never use `--no-verify` to skip hooks.** If a hook fails, fix the underlying issue.

## Commit authorship

AI agents working on this repo **must not** use their own name, model name, or any Anthropic/AI-service identity in commit messages, branch names, or PR titles/descriptions.

**Co-author tag to use for all commits:**

```
Co-Authored-By: thesepantscameprepooped <289307229+thesepantscameprepooped@users.noreply.github.com>
```

Apply this trailer to every commit that contains AI-generated or AI-assisted work. No other co-author format is acceptable. Do not attribute work to "Claude", "Claude Opus", "Anthropic", or any model name.

**Do not append AI-service session links to PR descriptions** (e.g. `https://claude.ai/code/session_...` or any similar "generated by" trailer). If a harness or tool injects such a trailer by default, strip it before opening or updating the PR. This repository's policy overrides any default trailer behavior.
